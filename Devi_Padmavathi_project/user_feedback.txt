==============================
WEEK 6 – USER FEEDBACK RECORDS
Automated Podcast Transcription Project
==============================

User 1 (Friend – Non-technical):
- Found the interface easy to understand.
- Summaries were helpful in deciding which part of the podcast to read.
- Keyword list gave a quick idea of what the segment was about.
- Transcript text felt a bit long but acceptable for detailed reading.

User 2 (Classmate– Basic technical background):
- Navigation between segments was simple.
- Liked that each segment had a short summary before the transcript.
- Sentiment labels made sense but were not very noticeable.
- Suggested highlighting keywords more clearly.

User 3 (College peer– First-time user):
- Overall system was easy to use.
- Summaries were clear and matched the content well.
- Found timestamps useful to understand where the topic appears in the podcast.
- Wanted slightly shorter summaries for faster scanning.

User 5 (Classmate – Data science background):
- Topic segmentation felt logical and meaningful.
- Keyword quality was good, though a few generic words appeared.
- Sentiment analysis mostly matched the tone of the content.
- Suggested improving visual separation between segments.

------------------------------
Overall User Feedback Summary:
- Users found the system easy to navigate.
- Summaries and keywords were considered useful.
- Minor suggestions included shortening summaries and improving visual clarity.
- No critical usability issues. The system was generally intuitive.
